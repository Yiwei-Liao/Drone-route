## 5. 风险评分验收

风险评分接口：`/api/risk`  
风险模型标识：`rule_based_prototype_not_safety_assessment`

当前风险评分是规则型原型，只用于数据探索和异常提示，不作为真实飞行安全评估。

### 电池余量风险

规则：

- 字段：`flight_tasks.battery_remaining`
- 条件：`battery_remaining < 45`
- 分值：+2
- 说明：原始字段名为“该架次结束后剩余的电池电量（V）”，不是百分比；阈值为原型经验阈值，需结合无人机型号和电池参数校准。

### 风速风险

规则：

- 字段：`flight_tasks.wind_speed`
- 条件：`wind_speed >= 8`
- 分值：+2
- 说明：单位来自原表 `m/s`，但阈值仍需结合无人机型号和作业规程确认。

### 航程风险

规则：

- 字段：`flight_tasks.flight_distance`
- 条件：`flight_distance >= 8`
- 分值：+1
- 说明：原始字段单位写为 `mk`，单位待确认，因此该风险只能作为候选提示。

### 航线高度风险

任务表规则：

- 字段：`flight_tasks.flight_height`
- 条件：`flight_height < 30` 或 `flight_height > 350`
- 分值：+1
- 说明：用于识别航线设定高度异常候选。

KML 规则：

- 字段：`routes.min_height`、`routes.max_height`
- 条件：`min_height < 10` 或 `max_height > 1000`
- 分值：+1
- 说明：KML 高度含义待确认，不能直接解释为真实离地高度。

### 距离机场风险

规则：

- 字段：`towers.distance_to_airport`
- 条件：`distance_to_airport > 5000`
- 分值：+1
- 说明：原始台账字段“与机场距离”单位待确认，因此只作为低置信候选风险。

### 杆塔匹配异常风险

规则：

- 字段：`route_tower_matches.match_confidence`
- 条件：`match_confidence == "low"`
- 分值：+2
- 说明：当前 3 条 KML 均为 high，未触发该风险。

## 7. 当前假设与边界

### 哪些结果来自真实数据

以下内容来自原始 Excel/KML 或基于真实坐标的确定性计算：

1. 巡检任务统计表中的任务记录、无人机型号、飞行高度、气象字段、飞行时长、飞行距离、电池电量、基站编号和基站经纬度。
2. 线路台账中的线路名称、杆号、杆塔经纬度、杆塔全高、档距、机场站点、与机场距离。
3. KML 中的航点经纬度、高度、速度、航向、云台俯仰角、转弯模式。
4. 航线长度、平均航点间距、航点到杆塔距离、最近杆塔匹配。
5. GeoJSON 图层中的杆塔点、基站点、KML 航线。

### 哪些结果是模拟值

当前项目未生成伪造 DEM、伪造通信覆盖、伪造机场坐标或伪造无人机能耗。

但以下内容属于规则推断或原型候选结果，不是严格仿真值：

1. 航线类型判断，如“单塔巡检（推断）”。
2. `/api/risk` 风险评分。
3. `lines.geojson` 中按台账顺序连接形成的线路候选几何。
4. 距离机场风险，因为原字段单位待确认。

### 哪些模块只是占位

1. `backend/taishan_pipeline/terrain.py`：DEM/DSM 地形采样接口占位。
2. `backend/taishan_pipeline/communication.py`：通信链路接口占位，目前只做距离计算。
3. `airports.csv`：只保存机场/机巢名称，坐标为空，等待真实机场坐标。
4. 前端当前为 Leaflet 二维地图原型，代码通过 GeoJSON API 组织，后续可替换或扩展为 CesiumJS。

### 当前是否接入真实 DEM/DSM 地形

否。

当前 `data/raw/dem/` 仅为预留目录，未接入真实 DEM/DSM。系统不会输出真实地形高程、坡度、遮挡或 AGL 离地高度。

### 当前是否接入真实通信模型

否。

当前只使用基站经纬度计算航点到基站的直线距离，不计算链路预算、天线方向图、覆盖强度、视距遮挡或信号质量。

### 当前是否接入真实无人机能耗模型

否。

当前仅基于任务表已有飞行距离、飞行时长、电池电量字段做规则提示，没有接入无人机型号参数、电池容量、功率曲线、载荷、风场或爬升能耗模型。

## 8. 下一阶段建议

### 优先影响验收的 bug / 数据缺口

1. 机场/机巢缺少经纬度  
   前端目前只能显示基站点，不能显示真实机场/机巢点。这影响“机场/基站展示”的完整性。建议优先补充机场/机巢经纬度。

2. 原始数据目录不符合规范  
   当前原始文件仍位于 `raw/`，虽然脚本支持回退扫描，但正式交付建议归档到 `data/raw/tasks`、`data/raw/ledger`、`data/raw/kml`。

3. 任务表两个时间字段同名  
   第 8、9 列均为“运行日志开始时间”，当前第 9 列只能暂记为结束时间候选。建议人工确认字段含义后再做时长校验。

4. 飞行距离单位异常  
   原字段为“无人机飞行巡检距离（mk）”，单位可能是 `km` 或其他口径。当前航程风险只能作为候选提示。

5. 部分杆塔缺少经纬度  
   当前 18 条杆塔记录缺少有效坐标，主要影响线路完整性和后续拓扑展示。

6. 线路拓扑未验证  
   `lines.geojson` 当前按台账行顺序连接杆塔，不能保证真实线路拓扑、分支和多回线路关系正确。

7. 风险评分阈值未经业务校准  
   电池、风速、距离、高度等阈值均为原型规则，不应作为真实安全评估。

### 视觉增强建议

1. 增加图层开关：杆塔、线路、KML 航线、基站、风险点可单独显示/隐藏。
2. 增加选中高亮：点击航线时同步高亮最近杆塔和匹配距离。
3. 优化中文标签避让：当前密集区域标签可能遮挡，建议按缩放级别显示。
4. 增加风险颜色映射：低/中/高风险使用统一图例。
5. 后续在数据逻辑稳定后，再接入 CesiumJS 三维地形视图；接入前必须先准备真实 DEM/DSM。
