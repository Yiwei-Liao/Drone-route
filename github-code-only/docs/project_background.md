# 项目背景说明

本项目面向电力线路无人机巡检场景，目标是基于现有巡检任务数据、线路台账和无人机 KML 航线，构建一个区域级巡检仿真与可视化原型。

## 已有数据

### 1. 巡检任务统计表

该表记录无人机巡检任务，包括巡检线路、杆塔编号、无人机型号、航线编号、航线高度、任务开始/结束时间、气象条件、飞行距离、飞行时长、电池余量、机场/基站编号和基站经纬度等信息。

该表用于分析任务执行情况、飞行距离、飞行耗时、电池余量、气象影响和基站关联关系。

### 2. 线路台账

该表记录线路和杆塔基础信息，包括线路名称、杆号、经度、纬度、杆塔全高、档距、机场站点和与机场距离等信息。

该表用于建立杆塔空间分布、线路走向和巡检对象。

### 3. KML 航线文件

KML 文件来自无人机巡检任务，包含航点经纬度、高度和可能的 DJI/MIS 扩展字段，例如速度、航向、云台角度和转弯模式等。

当前 KML 文件可能是单塔或局部巡检航线，不一定代表完整线路巡检航迹。

## 第一阶段目标

第一阶段不是构建完整数字孪生系统，而是验证以下问题：

1. 能否稳定读取 Excel 和 KML。
2. 能否提取杆塔、航点、机场和基站位置。
3. 能否计算航线长度、高度范围、航点间距和航向变化。
4. 能否判断 KML 航线与某个杆塔的空间关系。
5. 能否输出可复现的地图图件和指标表。

## 后续扩展方向

后续可逐步扩展：

1. 接入 DEM/DSM 地形数据，计算无人机离地高度 AGL。
2. 接入通信基站参数，分析无人机巡检过程中的通信链路风险。
3. 接入能耗模型，评估航程、电池余量和返航风险。
4. 搭建 Web/GIS 平台，实现线路、杆塔、航线和任务的交互式展示。
5. 接入知识图谱或智能体，用于巡检任务解释和风险提示。

## 边界说明

1. 当前没有真实 DEM/DSM 地形数据，因此不能输出真实地形风险结论。
2. 当前 KML 数量有限，因此不能直接代表完整区域巡检规律。
3. 当前电池和通信模型只能做经验分析，不能作为严格物理仿真结果。
4. 所有无法从数据验证的结论必须标记为假设或待验证。
